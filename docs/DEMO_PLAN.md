The final video must remain under three minutes, be publicly viewable on YouTube, include audio, and explain both the product and how Codex and GPT-5.6 were used.

- Open Founder Inbox.
- Show several synthetic leads that require the same manual workaround.
- State the problem: software waits for a feature request even when repeated intent is already visible.

### 0:20-0:50 - Evidence becomes a proposal

- Replay the typed event fixture.
- Show the repeated-intent detector crossing its threshold.
- Show GPT-5.6 interpreting the bounded evidence and drafting a capability contract.

### 0:50-1:25 - Codex generates inside a membrane

- Show the approved contract and constrained extension workspace.
- Show the generated diff.
- Explain where Codex accelerated implementation and where the entrant made the key product and authority decisions.

### 1:25-2:05 - Proof and a meaningful block

- Run unit, adversarial, permission, and replay checks.
- Show one deliberately unsafe behavior being rejected.
- Show the repaired version passing without weakening the contract.

### 2:05-2:35 - Human-approved evolution

- Review the proof bundle.
- Approve and install the capability.
- Use it on a new synthetic lead.

### 2:35-2:55 - Reversibility

- Disable and roll back the capability.
- Show the audit trail and restored prior state.

### 2:55-3:00 - Close

- "The model is creative. The system is sovereign. Software evolves only after it earns the right."
