## Current product map

```text
TypeScript Next.js App Router 15.3+ source using src/app
  -> bounded non-executing discovery
  -> Product Manifest + observation map + metric catalog

living install
  -> create-only browser observer + same-origin local collector
  -> hash-guarded install journal

ordinary UI activity
  -> privacy-minimized route/action/performance/friction/geometry events
  -> manifest-scoped hash-linked evidence
  -> deterministic workflow projection + privacy-safe event/metric matrix
  -> optional legacy detector diagnostics; never an AI gate

living improve --provider codex|api
  -> all verified events + all current metrics + bounded product context
  -> GPT-5.6 chooses the pattern and authors a strict evidence brief
  -> Living projects at most 3 affected UI files / 96 KB
  -> GPT-5.6 authors 1-8 exact edits to 1 existing file
  -> deterministic compilation + declared-authority denylist + static proof
  -> prepared evolution; host source unchanged

human review
  -> approve exact artifact + proof hashes
  -> engine applies only the bound postimage
  -> operator builds/reloads and verifies the host
  -> exact-postimage rollback remains available

explicit studio:sync
  -> Studio visualizes the same map, evidence, proposal, proof and receipts
  -> loopback-only development broker can invoke the selected provider/lifecycle
  -> Current vs Proposed remains display-only
```

The separate CRM and its simulator remain independent projects. Living derives its own map, verified behavior matrix, source candidates, and model requests. Simulator traces are a post-run oracle only and never tell Living what problem or patch to produce. Studio consumes an explicitly synchronized minimized snapshot rather than continuous live telemetry; its connected broker and the CLI share the same governed evolution ledger. The AI-first boundary is recorded in [ADR-001](ADR-001-AI-FIRST-DISCOVERY.md).

## Working now

1. Strict public contracts and local validation.
2. Bounded automatic mapping for the supported Next.js adapter.
3. Create-only installation, doctor validation, hash-journaled removal, and modified-file preservation.
4. Self-contained browser observation and same-origin single-process local collection without editing the host `package.json`.
5. Privacy-minimized workflow, performance, friction, viewport, and CSS-pixel geometry evidence.
6. A privacy-safe matrix containing all verified event sequences and all current workflow, performance, friction, viewport, visibility, geometry, movement, and timing metrics.
7. GPT-5.6 chooses one evidence-supported pattern and improvement hypothesis without a fixed detector gate, prescribed workflow, or hardcoded feature.
8. Legacy backtracking, correction, repeated-sequence, and dead/rage-click detectors remain as JSON/test diagnostics only.
9. Explicit Codex CLI and Responses API GPT-5.6 transports with no automatic fallback.
10. Two bounded model roles: AI-first behavior discovery/evidence interpretation and one-file source-patch authorship.
11. Deterministic target/preimage/anchor/diff/authority validation, proof, human approval requiring exact artifact and proof hashes, capture-verify/no-overwrite application, recovery, receipts, and exact rollback.
12. Terminal-first `install -> analyze -> improve -> approve/apply -> rollback`, with Studio as a visual companion.
13. Neutral fixtures and offline replay for credential-free deterministic verification.
14. Historical live CRM proposals across several detector domains and different GPT-authored targets, preserved as governed patch/apply/rollback evidence rather than the current AI trigger.
15. Evidence-first model context that preserves evidence-linked nodes and direct neighbors while preventing lexical context from becoming change-target authority.
16. Settled lifecycle status/list reads that are read-only; pending journals still enter locked recovery.

## Explicit non-claims

- The current adapter does not support arbitrary Node applications or arbitrary codebases.
- Automatic observation does not know business goals, outcomes, or causality.
- The local collector and Studio broker are not production authenticated, multi-instance, or remote services.
- Static schemas and denylist checks do not prove a generated patch is semantically correct, accessible, secure, buildable, or beneficial.
- Source application alone is not runtime verification.
- Living does not automatically capture or measure a post-change cohort.
- The separate CRM and simulator are not dependencies of Living core.
