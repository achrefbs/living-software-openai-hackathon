# Contributing

The submission window is short, so contributions must stay tightly aligned with the Founder Inbox vertical slice.

1. Read `AGENTS.md`, `PRIOR_WORK.md`, and `DECISIONS.md`.
2. Create a focused branch.
3. Keep generated changes small enough to understand and review.
4. Add or update tests for every behavior change.
5. Update `BUILD_LOG.md` and any affected decision or architecture record.
6. Run `npm test` before opening a pull request.

Do not add real user data, secrets, unlicensed assets, or code copied from the pre-hackathon prototype.
